# Адаптивная спецификация лендинга РУДН ИИ × X5 Tech

Макеты существуют только для двух ширин: **десктоп 1200** и **мобильная 320**. Всё между ними — правила из этого документа. Спецификация утверждается пользователем до начала вёрстки секций; изменения после утверждения — только явным решением.

## Диапазоны

| Диапазон | Раскладка | Правила |
|---|---|---|
| **320–767** | Мобильная (база 320) | Контейнер 100% ширины, боковые паддинги 20px. Шрифты, радиусы, отступы — фиксированные значения из мобильного макета. Фото и карточки тянутся по ширине (object-fit: cover, высоты фиксированы или aspect-ratio из макета). На 360–430 ничего не перевёрстывается — только растяжение. Пиксель-сверка — строго на 320; на 390 и 430 — смоук-контроль. |
| **768–1023** | Промежуточная | Контейнер 100% с паддингами 32px. Сетки карточек: 4 → 2 колонки. Двухколоночные блоки (Бюджет/Контракт, треки NLP/CV) — остаются 2 колонки. Hero — уже горизонтальный (текст слева, фото справа), но фото уже. Меню — ещё бургер (десктопная капсула не влезает по ширине с 4 пунктами + кнопка). |
| **1024–1199** | Десктопная | Контейнер 100% с паддингами 40px (как в макете: 1200 − 2×40 = 1120 контента). Полная десктопная раскладка, все сетки 4 колонки. Меню — десктопная капсула. |
| **≥ 1200** | Десктопная, фиксированный контейнер | Контентная колонка ровно 1200px по центру. Full-bleed элементы (лаймовые marquee-ленты, лавандовый фон секции X5 Tech, тёмный футер, фон шапки) тянутся на всю ширину окна. Hero-композиция (фото + блобы) остаётся в пределах 1200 — фоновая заливка секции белая до краёв. На 1440/1920 ничего не масштабируется — только поля растут. |

**Точка переключения мобильная/промежуточная раскладка: 768px. Бургер → капсула: 1024px.**

## Типографика между базами

Заголовки: десктоп H1 42px → мобильный размер из мобильного макета (уточняется по каждой секции при вёрстке); между базами — без fluid-магии, ступенька на 768. Основной текст 13–14px одинаков везде. Никаких clamp() без необходимости — предсказуемость важнее плавности, это же упрощает пиксель-сверку.

## Что считается багом, а что — легитимным растяжением (320–430)

- Легитимно: выросли боковые поля контента, фото шире, строки текста переносятся в других местах.
- Баг: горизонтальный скролл, обрезанный текст, наложение элементов, изменение размеров шрифта/радиусов, «прыжок» раскладки.
