# Бюджет производительности

Замеры — gzip, по `<script src>` в `out/index.html` (полифилы с `noModule` современные браузеры не грузят и в бюджет не входят).

## Базовая линия (пустой скаффолд, 2026-07-22)

| Сборщик | Все чанки | Полифилы (noModule) | Реально грузится |
|---|---|---|---|
| Turbopack (`next build`) | 186KB | внутри чанков | 186KB |
| **Webpack (`next build --webpack`)** | 167KB | 39.5KB | **~127KB** |

Решение: продакшн-сборка — webpack (`build` в package.json), dev — turbopack. Пол бюджета задаёт Next+React: ~127KB gz — меньше на этом стеке не бывает (для сравнения это цена выбора React; статический HTML рендерится без JS, гидрируются только 4 острова).

## Целевые значения

> **Про порог 125KB из плана.** В плане (Task 0.1 и Task 5.3) записано «≤ 125KB gz» — эта цифра выводилась из ожидания, что пол Next составит ~102–105KB. Фактический замер дал **127KB**, то есть 125KB недостижимы в принципе на этом стеке. Действующий порог — **150KB** из таблицы ниже; строки плана считаются устаревшими и заменяются этим документом.

- First Load JS (современный браузер): **≤ 150KB gz** (пол 127 + наш код островов ≤ 20KB + контент ~0: секции — Server Components)
- HTML главной: ≤ 40KB gz (база пустой страницы 3.1KB)
- Картинки hero-вьюпорта: ≤ 300KB суммарно
- Lighthouse: Performance ≥ 90 mobile / ≥ 95 desktop, Accessibility ≥ 95, CLS < 0.1
